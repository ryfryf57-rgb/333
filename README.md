# Mall Hunt — демо для Vercel

Готовый к деплою Vite-проект. Задеплоится на Vercel за 2 минуты и будет работать с HTTPS — значит камера и QR-сканер будут работать на реальном телефоне.

Приложение работает в **mock-режиме** (без бэкенда) — спавны генерятся в браузере, прогресс локальный. Идеально чтобы показать интерфейс коллегам/заказчику и протестировать камеру на мобильных.

## Деплой за 10 минут

### Если у тебя уже есть GitHub-аккаунт

1. Создай новый репозиторий на github.com (назови например `mall-hunt-demo`), сделай публичным или приватным — без разницы, Vercel умеет и так и так
2. Залей файлы этого проекта в репозиторий. Самый простой способ:
   ```bash
   # Распакуй архив, зайди в папку
   cd mall-hunt-vercel
   git init
   git add .
   git commit -m "initial"
   git remote add origin https://github.com/ТВОЙ_НИК/mall-hunt-demo.git
   git branch -M main
   git push -u origin main
   ```
   Или просто зайди на страницу репозитория и через «Upload files» закинь все файлы мышкой.

3. Открой **https://vercel.com** → «Sign up» → войди через GitHub
4. На дашборде Vercel → кнопка **«Add New…» → «Project»**
5. Найди в списке свой репозиторий `mall-hunt-demo` → **Import**
6. Vercel сам определит что это Vite-проект. Ничего менять не нужно. Жми **Deploy**.
7. Через 1-2 минуты получишь ссылку типа `https://mall-hunt-demo-xxxx.vercel.app`

### Если без GitHub

Ещё проще — скачай **Vercel CLI**:
```bash
npm install -g vercel
cd mall-hunt-vercel
vercel
```
Подтверди email, ответь на несколько вопросов (все ответы по умолчанию) → через минуту получишь ссылку.

## Что тестировать на телефоне

1. Открой полученную Vercel-ссылку на телефоне (просто скопируй в Safari/Chrome)
2. Нажми **«Войти через Telegram»** (не нужен реальный Telegram — в mock-режиме это работает как скип)
3. Пройди онбординг
4. На карте ТЦ дождись появления существа (≈8 секунд)
5. Нажми на зону → откроется AR-экран
6. Нажми **«Включить AR-режим»** → телефон попросит доступ к камере → дай разрешение
7. Должна включиться задняя камера, появится существо поверх реального мира
8. Свайп снизу вверх по экрану — бросок шара
9. Проверь остальные вкладки: коллекцию, квесты, профиль

## Известные ограничения mock-режима

- Каждое устройство видит свой мир (нет общего сервера)
- Прогресс теряется при перезагрузке браузера (локальное хранилище)
- QR-сканер на реальной наклейке не даст поимки — это требует бэкенд с реальными `qr_secret`
- Отображается бейдж **«offline demo»** сверху экрана — это нормально, так и должно быть

## Что делать когда захочется подключить реальный бэкенд

В будущем, когда развернёшь бэкенд (например на Railway):

1. В Vercel: **Project Settings → Environment Variables**
2. Добавь переменную `VITE_API_URL` = `https://твой-бэк.railway.app`
3. Redeploy

После этого приложение перестанет работать в mock-режиме и начнёт ходить в реальный сервер.
